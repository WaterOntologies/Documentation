This is a specification for defining a scientific lens: a filter for varying the linksets that are activated to achieve associations between datasets that meet a set scientific working assumptions. The specification defines the attributes that need to be stated in order that the Open PHACTS Identity Mapping Service can apply the lens to control the active linksets.

Intended audience

This lens specification is intended for application developers and data providers. First so that applications can understand and enable users to apply lenses. Second to enable the creation of lenses. Finally to enable linksets to be associated with lenses.

Introduction

The Open PHACTS platform [[OPS-ARCH]] relies on data and the interlinks published by a variety of sources. For example, details of chemicals are derived from ChemSpider, ChEMBL, and DrugBank. This specification defines scientific lenses which enable the relationships between datasets to be varied to meet the scientific assumptions required. For example, the ChemSpider record for imatinib mesylate can be related to the DrugBank record imatinib in several ways: first because one is the salt-form of the other and secondly because they share the same drug name, viz Gleevec. There are circumstances where scientists would like to keep the records separate, i.e. where they are concerned about analysing chemcicals based on their weight, whereas in other cases they would like the records to be integrated as they are about the same drug.

The Open PHACTS project has produced a set of guidelines aimed at data providers for publishing their data within the OPS system [[!OPS-RDF]]. The RDF guide provides details about modelling your data as RDF. In another specification, we have stated the metadata that should be published to describe the dataset and the links to other datasets [[!OPS-METADATA]]. This specification builds upon both of these.

This specification declares the properties that should be included in the definition of a scientific lens. The properties should enable user faciing applications to support scientists in choosing between lenses. Equally, the properties should enable the application of lenses to switch on/off linksets in a given scenario.

Document Conventions

RDF Syntax and Namespaces

All examples in this document are written in the Turtle RDF syntax [[TURTLE]]. Throughout the document, the following namespaces are used: 

@prefix dcterms: <http://purl.org/dc/terms/> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .

Furthermore, we assume that the empty prefix is bound to the base URL of the current file like this: 

@prefix : <#> .

This allows us to quickly mint new identifiers in the local namespace.
